# ResNet18 迁移学习学习方案

## 1. 学习目标

这部分的目标不是“看懂代码就结束”，而是你要通过亲手修改、补全、验证，真正掌握下面这些能力：

- 能解释什么是迁移学习，为什么预训练模型有用
- 能独立搭建一个 `ResNet18` 迁移学习分类脚本
- 能区分“冻结训练”和“微调训练”
- 能完成训练、验证、保存、单图推理这一整条闭环
- 能自己改超参数、改训练策略、改输出内容，并判断改动是否合理

最终标准不是“我好像懂了”，而是：

- 你能自己从零重写一个简化版迁移学习脚本
- 你能解释每个关键设计背后的原因
- 你能自己定位常见报错和训练异常

---

## 2. 学习原则

这次学习采用项目导向方式，原则只有 3 条：

- 先跑通，再理解，再优化
- 多改小地方，多做对比实验，不空想
- better done than perfect，先拿到结果，再逐步打磨

你接下来不要只“读代码”，而要把代码当成练习题来做。

## 6. 第二阶段：按模块拆开学

这一阶段不要“整体看一遍就过去”，要按模块拆开，一个点一个点吃掉。

### 6.1 数据部分

重点函数：

- `build_dataloaders()`
- `train_transform`
- `val_transform`

你要弄懂：

- 为什么这里直接用 `ImageFolder`
- 为什么训练集和验证集的 transform 不一样
- 为什么预训练模型常用 ImageNet 的 mean/std
- 为什么验证集不能做随机增强

练习任务：

- 把训练集和验证集的类别映射打印得更清楚
- 在脚本里打印一张 batch 图片的最小值、最大值、shape
- 暂时把 `train_transform` 和 `val_transform` 改成一样，观察结果和意义

### 6.2 模型部分

重点函数：

- `build_model()`
- `set_trainable_layers()`

你要弄懂：

- `resnet18(weights=...)` 到底帮你做了什么
- `model.fc.in_features` 是什么
- 为什么最后改成 `nn.Linear(in_features, 2)`
- 为什么阶段 A 只训练 `fc`
- 为什么阶段 B 解冻 `layer4 + fc`

练习任务：

- 打印 `model`
- 打印 `model.fc`
- 统计总参数量和可训练参数量
- 把阶段 B 从 `layer4 + fc` 改成“只解冻 fc”，比较结果
- 把阶段 B 改成“解冻全部层”，比较结果

### 6.3 训练部分

重点函数：

- `train_one_epoch()`
- `evaluate()`
- `save_checkpoint()`

你要弄懂：

- 训练循环 5 步在迁移学习里还是不是一样
- `model.train()` 和 `model.eval()` 什么时候切换
- 为什么验证时要 `torch.no_grad()`
- 为什么按 `val_acc` 保存最优模型，而不是按 `train_loss`

练习任务：

- 把每个 epoch 的日志打印格式改成你更容易看懂的样子
- 增加“当前 best acc”打印
- 保存 `last checkpoint`
- 额外打印每个阶段用了多少个 epoch

### 6.4 推理部分

重点函数：

- `load_checkpoint()`
- `predict()`

你要弄懂：

- 为什么单图推理要用验证集预处理
- 为什么要 `unsqueeze(0)`
- 为什么输出还要过 `softmax`
- `idx_to_class` 为什么要从 `class_to_idx` 反转回来

练习任务：

- 输出 top-2 概率
- 增加“预测是否高置信度”的提示
- 支持输入一个文件夹，对多张图批量推理

## 8. 第四阶段：必须亲手做的对比实验

这一阶段很重要。不要只改代码，要做对比实验。

### 实验 1：预训练 vs 非预训练

对比：

- `weights=ResNet18_Weights.DEFAULT`
- `weights=None`

你要观察：

- 哪个收敛更快
- 哪个准确率更高
- 小数据集上差距是否明显

### 实验 2：冻结训练 vs 全量微调

对比：

- 只训 `fc`
- `layer4 + fc`
- 全量训练

你要观察：

- 哪种最稳
- 哪种最容易过拟合
- 哪种最适合当前数据规模

### 实验 3：不同输入预处理

对比：

- 正常使用 `ImageNet Normalize`
- 去掉 `Normalize`
- 把训练集和验证集都用同一套 transform

你要观察：

- 准确率变化
- 训练稳定性变化
- 为什么预处理和预训练模型是绑定的

## 10. 你必须掌握的常见坑

### 输入尺寸不对

典型现象：

- shape 报错
- 准确率很差

你要知道：

- `ResNet18` 预训练通常配套 `224x224`
- 当前项目里 transform 已经帮你处理好了

### 忘了切换 `eval()`

典型现象：

- 验证结果不稳定

你要知道：

- 验证前要 `model.eval()`
- 训练前要 `model.train()`

### 忘了 `no_grad()`

典型现象：

- 验证阶段显存占用变大
- 推理变慢

### checkpoint 和模型结构不匹配

典型现象：

- `load_state_dict` 报错

你要知道：

- 推理时模型结构必须和训练保存时一致
- 这里二分类头必须还是 `Linear(in_features, 2)`

### 类别映射错了

典型现象：

- 预测名称和实际类别对不上

你要知道：

- `class_to_idx` 必须保存
- 推理时要正确反转成 `idx_to_class`
